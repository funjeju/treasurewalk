# 04 · 기능 & 로드맵

## 1. 최대 프레임워크 — 「가족 보물찾기 플랫폼」 6기둥
끝까지 커졌을 때의 전체 구조. 모든 기능은 이 6칸 중 하나에 들어간다. 각 기둥은 enum/플러그인으로 쪼개져 있어 **칸을 늘리는 게 아니라 값을 켜는 것**으로 확장된다.

```
① 보물 생성   지도핀 · 로드뷰 · EXIF추출 · AI추천POI / 단일·체인·도감·시즌
② 탐험·검증   만보기 · 실시간지도 · 지오펜스 · 사진/EXIF인증 · 촬영시각 · (AR)
③ 보상 경제   고정 · 거리비례 · 스트릭 / 코인·젬·도감·레벨·뱃지
④ 사회 루프   가족피드 · 인증서 · 카톡연동 · 형제 협동/경쟁
⑤ 안전·거버넌스  허용구역 · 위험구역제외 · 위치 OFF기본 · 부모통제·동의
⑥ 데이터·인사이트  발견로그 · 걸음리포트 · 부모 대시보드
```

## 2. 페이즈 사다리 — 각 기둥에서 몇 칸 켜나

| 기둥 | **P1 (지금)** | P2 | P3 | P4 |
|---|---|---|---|---|
| ① 생성 | 지도핀+반경+힌트사진 | EXIF·로드뷰 | 미션 체인·도감 | AI POI 추천 |
| ② 검증 | 실시간 지오펜스 진입 | 사진 EXIF·촬영시각 | AR 연출 | — |
| ③ 보상 | 고정 금액 + 코인 | 거리비례·스트릭·젬 | 도감·레벨·뱃지·상점 | 시즌 경제 |
| ④ 사회 | 카톡 **공유** 인증서 + 가족피드 | 카톡 자동발송 | 형제 협동/경쟁 랭킹 | 가족 간 미션 교환 |
| ⑤ 안전 | 위치 OFF기본·부모확정·반경 | 위험구역 제외 | 자녀 독립 로그인 | 정식 동의·사업자 |
| ⑥ 데이터 | 발견 로그 | 걸음 리포트 | 부모 대시보드 | 인사이트 |

> Claude Code: **P1 열만 구현.** 나머지 칸은 `docs/03` 스키마의 enum/nullable로 자리만.

## 3. ⭐ Phase 1 — 정확한 범위 (이것을 완성하라)

### 3.1 부모 플로우 (구현 화면 4)
1. **구글 로그인** → 최초 1회 **가족 생성**(이름·기본 언어) + **자녀 프로필 추가**(이름·연령티어).
2. **보물 숨기기:** 지도에서 핀 1개(`MAP_PICK`) → 반경 슬라이더(30~50m) → 힌트 사진 1장 업로드(Storage) → 용돈 금액 입력 → `보물 숨기기`(treasure status:active).
3. **대시보드:** 활성 보물 목록 + 자녀별 발견/요청 현황.
4. **지급 확정:** claim이 `REQUESTED`면 인증서 확인 → `지급완료`(status:PAID).

### 3.2 아이 플로우 (구현 화면 3)
1. **게임 월드 지도:** 보물 핀 + 힌트 썸네일 + 내 위치 점 + **오늘 걸음 수** + HUD(코인·레벨·스트릭). 위치 OFF면 켜기 안내.
2. **탐험(보물찾기 모드):** `watchPosition` + **근접 미터(뜨거워/차가워)** + 거리 표시. 반경 진입 시 발견 트리거.
3. **발견 연출 + 요청:** 상자 열기(Rive) + 코인 분수(Lottie) + 팡파르 → `용돈 요청하기` → **인증서 이미지 생성** → OS 공유시트로 가족 카톡 전송(`KAKAO_SHARE`).

### 3.3 인증서(Certificate) 내용
탐험가 이름 · 발견 시각 · 위치 미니맵 핀 · 오늘 걸음 수 · 요청 금액 · 짧은 발견ID. (클라이언트에서 canvas/SVG→PNG 생성 후 Storage 업로드 또는 공유.)

### 3.4 메타(가벼운 버전)
- **보물 도감:** 발견한 보물이 채워지는 그리드(P1: 단순 목록형도 OK).
- **레벨/코인/스트릭 HUD:** 발견 시 적립·연출.
- **가족 피드:** "○○ 보물 발견!"(activity 컬렉션).

### 3.5 횡단 기능 (P1 필수)
- **i18n:** ko/en 최소 2개 (`docs/06`).
- **다크모드 토글:** system/light/dark (`docs/06`).
- **반응형 + 접근성 floor:** 모바일 우선, 키보드 포커스, reduced-motion 존중.

## 4. 상태 흐름 (claim)
```
보물 발견(지오펜스 진입)  → claim.status = FOUND
        ↓ 아이가 [용돈 요청하기]
인증서 생성              → REQUESTED  → 가족 카톡 공유(KAKAO_SHARE)
        ↓ 부모 확인 후 용돈 지급
부모 [지급완료]          → PAID  (코인/XP 적립·도감 반영은 FOUND 시점)
```
> `PAID` 확정은 **부모 수동**. "보냈는데 받았나?" 모호함 제거.

## 5. P1에서 의식적으로 빼는 것 (빼도 안 부서지는 이유)
- 사진 EXIF 인증 → `verification` 배열에 값 추가만 하면 P2.
- 거리비례·스트릭 보상 → `reward.type` 교체만.
- 카톡 자동발송 → `notifyChannel` enum에 `KAKAO_AUTO` 자리 있음. **P1은 공유 방식만.**
- 백그라운드 위치 → P1은 포그라운드만(`docs/03 §5`).
